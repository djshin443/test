## 2. 잘 된 점 (강점)

### 2.1 문서 체계

- **4단계 분류 체계**가 명확: 설계 → 운영 → 교육 → 변경이력
- **대상별 문서 분리**: 기술자용(PRESENTATION.md) / 비기술자용(PRESENTATION_EASY.md)
- **마스터 문서(SYSTEM_LOGIC.md) 중심 구조**: 전체 시스템을 한 문서에서 파악 가능

### 2.2 운영 가이드 품질

- **DETECTION_LOGIC.md**가 특히 우수:
  - NIST SP 800-94, Gartner 보고서 등 업계 근거 제시
  - 오탐/미탐 트레이드오프 설계 철학 명문화
  - 실제 사고 사례 (2016 호주 Census) 인용
  - 임계값 설계 근거를 구간별로 설명
- **OPERATION_GUIDE.md**: 1번 vs 2번 메뉴 비교표, 체크포인트 재개 로직이 실전적
- **BACKUP_DEPLOY.md**: 백업→복원→비교→배포까지 완결된 워크플로우

### 2.3 교육 자료

- 인터랙티브 HTML 10개가 시각적으로 매우 효과적
- "공항 보안 검색대", "미술 복원가", "형사" 등 비유가 비기술자 이해에 적절
- 지도학습 vs 비지도학습 비교가 잘 구성됨

### 2.4 기술적 완성도

- 버그 67건을 CRITICAL/HIGH/MEDIUM으로 분류 추적
- 라벨 매핑 289개 중앙 관리 구조
- 보안: HMAC-SHA256, RestrictedUnpickler, 원자적 I/O 등 다층 방어

### 2.5 사이트 기능

- 검색 기능 (API 기반 + 키보드 단축키)
- PDF 다운로드 (개별 + 전체 합본)
- 반응형 CSS 기본 대응

## 3. 개선점 (아쉬운 점)

### 3.1 컨텐츠 일관성 문제

| # | 문제 | 위치 | 설명 | 수정 방안 |
|---|------|------|------|-----------|
| 1 | **피처 수 혼동** | SYSTEM_LOGIC, PRESENTATION | "30차원 특징"이라 하다가 FEATURE_SPEC에선 "59피처". 독자가 처음 읽으면 혼란 | 모든 문서에서 "59피처(30기본+3확장+26페이로드)" 통일 |
| 2 | **버전 불일치** | PRESENTATION.md, PRESENTATION_EASY.md | v1.2.7로 표기되어 있으나 시스템은 v1.2.8 | 발표 자료를 최신 버전에 맞춰 업데이트 |
| 3 | **상대적 시간 표현** | ROADMAP.html | "대표이사 시연 다음 주 예정" — 날짜가 지나면 의미 없어짐 | 절대 날짜로 변경 (예: 2026-04-03) |
| 4 | **사이드바 이름 혼재** | 사이드바 전체 | "BACKUP DEPLOY", "DETECTION LOGIC", "OPERATION TOOLS"는 영문 그대로, 나머지는 한글 | 전체 한글로 통일 |
| 5 | **발표 자료 중복** | PRESENTATION vs DETECTION_GUIDE | 상당 부분 내용이 겹침 | 역할 구분 명확화 또는 하나로 통합 |

### 3.2 문서 구조 문제

| # | 문제 | 설명 | 수정 방안 |
|---|------|------|-----------|
| 1 | **교육 자료 섹션 과부하** | 14개 문서가 한 카테고리에 뒤섞임 | "개념 교육" / "인터랙티브 시각화" / "발표 자료"로 소분류 |

| 3 | **트러블슈팅 분산** | OPERATION_GUIDE, TRAINING_LOGIC, MODULE_GUIDE에 각각 흩어져 있음 | 별도 `TROUBLESHOOTING.md`로 통합 |
| 4 | **용어집(Glossary) 부재** | EDUCATION.md에 일부 설명이 있지만 한 곳에서 참조 불가 | 별도 용어집 페이지 추가 |
| 5 | **CHANGELOG ↔ BUGFIX 중복** | 두 문서의 내용이 상당 부분 겹침 | CHANGELOG에 요약만, BUGFIX에 상세 — 역할 분리 명확화 |

### 3.3 UI/UX 개선점

| # | 문제 | 현재 상태 | 개선 방안 |
|---|------|-----------|-----------|
| 1 | **페이지 내 목차(TOC) 없음** | SYSTEM_LOGIC.md 같은 긴 문서에서 섹션 이동 불편 | 우측 고정 TOC 사이드바 추가 |
| 2 | **이전/다음 네비게이션 없음** | 문서 간 순차 탐색 불가 | 문서 하단에 "← 이전 \| 다음 →" 링크 |
| 3 | **최종 수정일 미표시** | 문서가 언제 기준 정보인지 알 수 없음 | 각 문서 상단에 최종 수정일 표시 |
| 4 | **PDF 생성 취소 불가** | 오버레이가 뜨면 취소 방법 없음 | 취소 버튼 추가 |
| 5 | **모바일 사이드바** | 모바일에서 전체 너비 차지 | 햄버거 메뉴 + 오버레이 방식 |
| 6 | **코드 블록 복사 버튼 없음** | 명령어를 수동으로 드래그해서 복사 | 코드 블록 우상단에 복사 아이콘 추가 |
| 7 | **다크모드 미지원** | 사이드바만 다크, 본문은 라이트 고정 | 다크모드 토글 추가 |

## 4. 로드맵 추가 제안

### 4.1 Phase 2 (현재)에 추가할 것

| 항목 | 설명 | 시연 효과 |
|------|------|-----------|
| **방화벽 자동 연동 스크립트** | blocked_ips.csv 변경 감지 → iptables/nftables 자동 업데이트 | "탐지→차단"이 자동으로 이어지는 것을 보여줄 수 있음 |

| **벤치마크 결과 문서** | Confusion Matrix, Precision/Recall/F1, FPR 실측값 | 기술 신뢰성 증명의 핵심 근거 |

### 4.2 Phase 3에 추가할 것

| 항목 | 설명 | 필요성 |
|------|------|--------|
| **실시간 알림 (Slack/Teams/Telegram)** | 웹훅 기반 즉시 알림. 이메일만으로는 긴급 공격 대응 불가 | 운영 필수 |
| **SIEM 연동 포맷** | Splunk/ELK(Elasticsearch)로 탐지 이벤트 내보내기 | 기업 도입 시 거의 필수 요구사항 |

### 4.3 Phase 4에 추가할 것

| 항목 | 설명 | 기대 효과 |
|------|------|-----------|
| **GeoIP 시각화** | 공격 출발지를 세계 지도에 표시 | 대시보드 시각적 임팩트 |
| **모델 A/B 테스트 (Shadow Mode)** | 새 모델 배포 전 기존 모델과 병행 비교 | 성능 하락 사전 방지 |
